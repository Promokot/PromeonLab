# 3.3 Технологическое обеспечение

> Статус: ЧЕРНОВИК (2026-06-05), на вычитке у Макса. План и решения – plan-3.3.md.

Технологическое обеспечение охватывает аппаратную платформу, средства реализации и порядок развёртывания приложения. Определяющее свойство решения – автономность: приложение исполняется непосредственно на гарнитуре виртуальной реальности, эксплуатация не требует персонального компьютера и сетевого подключения, а все рабочие данные размещаются в локальном хранилище устройства.

Целевой платформой является автономная гарнитура Meta Quest 3, заявлена также совместимость с предыдущей моделью линейки Meta Quest 2. Устройства этого класса сочетают встроенный вычислительный модуль под управлением операционной системы на базе Android, раздельный вывод изображения для каждого глаза и пару контроллеров с пространственным отслеживанием [ИСТ: характеристики автономных VR-гарнитур]. Этих средств достаточно для полного цикла работы – от импорта модели до выгрузки экспортного архива.

Приложение реализовано на игровом движке Unity версии 6000.3.7f1 с конвейером рендеринга Universal Render Pipeline, рассчитанным в том числе на графические процессоры мобильного класса [ИСТ: документация Unity/URP]. Доступ к средствам виртуальной реальности построен на открытом стандарте OpenXR [ИСТ: стандарт OpenXR]. Стандарт не привязывает приложение к программным средствам конкретного производителя гарнитур, поэтому решение потенциально способно исполняться и на других автономных устройствах, однако работоспособность за пределами целевой платформы экспериментально не проверялась. Обработка пользовательского ввода и взаимодействие с виртуальными объектами опираются на пакеты XR Interaction Toolkit и Input System. Состав технологической платформы приведён на Рисунке 3.11.

Сборка выполняется под платформу Android: исходный код транслируется средством IL2CPP в нативный код архитектуры ARM64, вывод графики осуществляется через Vulkan, результатом служит установочный пакет APK для размещения на гарнитуре. Разработка и отладка велись на персональном компьютере в редакторе Unity. Функциональная проверка выполнялась на гарнитуре, подключённой к компьютеру в режиме трансляции, когда программа исполняется в редакторе, а изображение и ввод передаются на устройство.

Для эксплуатации достаточно гарнитуры с парой контроллеров, свободного места в локальном хранилище под импортируемые модели, сохраняемые сцены и экспортные архивы, а также свободного пространства вокруг пользователя для безопасного перемещения. Подключение к сети и внешние службы не используются.

Ограничением текущей версии является объём проведённой проверки. Функциональное тестирование осуществлялось только на гарнитуре Meta Quest 3, подключённой к компьютеру. Автономная сборка полной проверки непосредственно на устройстве не проходила, поэтому при самостоятельном исполнении возможна нестабильность, в том числе в части целостности сохраняемых данных. Совместимость с Meta Quest 2 и иными гарнитурами со средой OpenXR экспериментально не подтверждена. Перечисленные позиции определяют направление дальнейшей доработки приложения.
