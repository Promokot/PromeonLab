## Проверка по коду (сводка фактов)

- Модуль из четырёх файлов: SceneExporter (267 строк), SceneBundle (59), события SceneExportRequestedEvent (8) и SceneExportedEvent (10). Панель – ExportPanel (125), модуль пользовательской панели на вкладке exporter панели навигации.
- SceneExporter – объект корневой области жизни (IStartable): подписан на SceneExportRequestedEvent, по завершении публикует SceneExportedEvent (путь, успех, сообщение). Запуск только через события; панель дополнительно обращается к BuildTargetPath для подписи пути (имя очищается от недопустимых символов, пустое заменяется на export).
- ExportPanel: поле имени файла (по умолчанию подставляется имя сцены), подпись вычисленного пути, имя активной сцены, кнопка экспорта (блокируется на время операции), строка статуса из события результата. Реагирует на SceneContextChangedEvent.
- Порядок экспорта (RunExportAsync): проверка SceneContext.HasScene → на главном потоке снимок: CaptureSnapshot графа + CaptureForExport анимационных данных (в песочнице анимационная часть опускается) → разрешение источников → чистая статическая BuildBundle → JsonUtility → запись ZIP на фоновом потоке (Task.Run). Ошибки перехватываются, публикуется событие с неуспехом.
- Разрешение источника (Resolve): по AssetRef запись находится в AssetRegistry, относительный SourceRef превращается в абсолютный путь, проверяется существование файла.
- BuildBundle – чистая функция (вход: SceneData, SceneAnimationData, делегат разрешения, метка времени; выход: SceneBundle + список файлов-источников). Нода: трансформация, позы костей, блок анимации контейнера владельца (кадры, интерполяция, цикл, треки; треки костей – префикс bone:). Импортированные с существующим источником → models/{assetId}.{ext} или textures/… (Reference → textures), дедупликация по пути в архиве; встроенные и потерянные источники → geometryMissing.
- SceneBundle – внешняя схема scene.json (schemaVersion 1, метка времени, ссылка на сцену, fps, ноды). Односторонний снимок, НЕ формат обратного импорта. Переиспользует BonePose и AnimKeyData.
- WriteZipBundle: без Unity API (безопасно на фоновом потоке), ZipArchive из System.IO.Compression; scene.json текстом, источники потоково. Путь: Documents/{Application.productName}/{имя}.zip – вне локального хранилища (результат для пользователя).
- Сообщение результата: путь + счётчик нод без геометрии.
- ⚠ EditMode-тесты модуля (SceneBundleTests, SceneExporterBuildTests, SceneExporterZipTests) в тексте НЕ описываются (решение F5).
- ОГОВОРКА МАКСА (2026-06-05): формат выведен, но для чтения пакета сторонним редактором (например, Blender) нужен мост-аддон на принимающей стороне; его разработка выходит за рамки задач ВКР, поэтому не выполнялась. Нейтрально, как ограничение и направление развития.

## Структура текста (как написано)

1. Вводный абзац: экспорт – завершающий этап работы со сценой; состав модуля (панель + экспортёр); событийная связка запрос/результат.
2. Панель экспорта: поля, обновление по смене сцены, путь через BuildTargetPath, блокировка кнопки, статус. [Рисунок 3.Э1 СКРИНШОТ; Листинг 3.Э1 фрагмент ExportPanel; полный – Б.Э1]
3. Снимок состояния: HasScene, CaptureSnapshot + CaptureForExport на главном потоке, разрешение источников через AssetRegistry. [полный SceneExporter – Б.Э2]
4. BuildBundle: чистая функция, перенос нод, отбор и дедупликация источников, geometryMissing. [Листинг 3.Э2 фрагмент]
5. Внешняя схема SceneBundle: состав корня, адресация треков, переиспользование BonePose/AnimKeyData, односторонность. [Листинг 3.Э3 целиком, в Б не дублируется]
6. Запись ZIP: фоновый поток, ZipArchive [ИСТ], путь в Documents, сообщение результата. [Листинг 3.Э4 фрагмент; Рисунок 3.Э2 СКРИНШОТ состава пакета]
7. Финал: собственная схема ≠ обменный формат → мост-аддон (пример Blender) вне рамок работы, направление развития; закрытие рабочего цикла приложения.
