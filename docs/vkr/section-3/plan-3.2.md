## 3.2.1 Архитектурный каркас приложения
- Содержание: шина событий EventBus (Publish/Subscribe, типизированные struct-сообщения – здесь «легализуются» все *Event, упомянутые в 3.1); контейнер зависимостей VContainer и иерархия областей (RootLifetimeScope – время жизни приложения, сценные области – время жизни режима; правило «ребёнок зависит от родителя, не наоборот»); SceneContext как доступ к сценным сервисам; режимы: ModeOrchestrator + ModeTransitionGraph, переходы через SceneTransitionRunner (затемнение HeadFade, одиночная загрузка сцены, порядок ModeExiting → ModeChanged – обратная отсылка к автосохранению из 3.1.4); песочница – один абзац.
- Листинги: EventBus.cs (целиком, 28); ModeOrchestrator.cs (целиком, 49); фрагмент регистрации из RootLifetimeScope.cs (полный – Приложение Б).
- Рисунки: схема иерархии областей DI (СХЕМА – генерирую); схема графа переходов режимов (СХЕМА).
- Возможно Таблица 3.1: область → время жизни → ключевые сервисы (ВОПРОС: нужна ли первая таблица?).

## 3.2.2 Система пространственных интерфейсов
- Содержание: SpatialPanel и режимы крепления (BodyLocked / WorldFixed / Free, биллборд); регионная модель: PanelRegionRouter + NavBarConfig + RegionMember/RegionNavButton (навбар «один регион – одна активная панель»); UserPanel: перемещение за драг-хэндл, замочек-фиксация (решение 5 – рассказываем подробно); VrKeyboard кратко; перечень панелей приложения одним абзацем-списком (детали – в функциональных пунктах, без номеров).
- Листинги: фрагмент SpatialPanel.cs (режимы крепления); фрагмент PanelRegionRouter.cs (маршрутизация открытия). Полные: SpatialPanel, PanelRegionRouter, UserPanel → Приложение Б.
- Рисунки: СКРИНШОТ UserPanel с навбаром и открытой панелью; СХЕМА регионной модели.

## 3.2.3 Главное меню и управление сценами
- Содержание: сценарий запуска – MainMenuPanel, ScenePickerPanel (создание/выбор/удаление сцены); открытие сцены: загрузка SceneData (отсылка к 3.1) → построение рантайм-графа SceneGraph/SceneNode; OutlinerPanel – отображение иерархии.
- Листинги: фрагмент SceneGraph.cs (построение графа из данных, CaptureSnapshot – пара к снимку из 3.1.4). Полный SceneGraph → Приложение Б.
- Рисунки: СКРИНШОТ главного меню; СКРИНШОТ пикера сцен; СКРИНШОТ аутлайнера (можно объединить 2 в один кадр, если влезает).

## 3.2.4 Библиотеки ассетов и импорт моделей
- Содержание: три библиотеки и AssetRegistry; конвейер импорта: системный пикер (SimpleFileBrowser – стороннее), FilePickedEvent → выбор импортёра по расширению (glTFast для glb/gltf, изображения), ImportWizardPanel → копирование источника + запись в библиотеку + ThumbnailRenderer (офскрин-миниатюра); галерея AssetBrowserPanel; размещение: AssetSpawnRequested → AssetSpawner → реестр билдеров (Object/Rig/Reference), build-once/restore-many (отсылка к шаблону из 3.1.2 – момент «build» создаёт шаблон, «restore» применяет); editor-бейк рецептов встроенных ассетов (BuiltinRecipeBaker, с пометкой editor-only); словесная отсылка вперёд для скелетных моделей (без номера пункта).
- Листинги: фрагмент ImportPipeline.cs (выбор импортёра); фрагмент AssetSpawner.cs или целиком (50 строк); фрагмент ThumbnailRenderer.cs; фрагмент BuiltinRecipeBaker.cs (бейк). Полные: ImportPipeline, AssetEntityBuilderRegistry, ThumbnailRenderer, BuiltinRecipeBaker → Приложение Б.
- Рисунки: СХЕМА конвейера импорта; СКРИНШОТ браузера ассетов с миниатюрами; СКРИНШОТ мастера импорта; ПАРНЫЙ РИСУНОК «модель в Blender ↔ модель в приложении» (готовит Макс).

## 3.2.5 Построение скелетных структур
- Содержание: только построение (взаимодействие с костями – в следующем пункте): извлечение скелета при импорте RigDefinitionExtractor (иерархия костей → RigDefinition в шаблоне); восстановление при спавне: RigEntityFabricator.BuildProxyRig – прокси-кость на каждую кость скелета, BoneFollower связывает прокси с костью скинированного меша; ProxyRigRuntime – владение прокси-ригом; применение сохранённых поз BonePoses (отсылка к 3.1.2); планирование селекторных боксов (BoneSelectorBoxPlanner, boneColliderDepth из шаблона).
- Листинги: RigDefinitionExtractor.cs (целиком, 19); BoneFollower.cs (целиком, 37); фрагмент BuildProxyRig из RigEntityFabricator.cs. Полные: RigEntityFabricator, ProxyRigRuntime → Приложение Б.
- Рисунки: СКРИНШОТ модели с прокси-костями; СХЕМА прокси-рига (скелет меша ↔ прокси-объекты ↔ BoneFollower).
- ⚠ Без IK, без «ограничителей».

## 3.2.6 Взаимодействие с объектами и костями
- Содержание: селекция: SelectionManager (single-select) + SelectionChangedEvent, реакции (подсветка SelectionVisualSync/QuickOutline – стороннее ядро, наш синк); прямое взаимодействие XRPromeonInteractable (tap – выбор, удержание триггера – вращение, удержание грипа – перемещение); гизмо: GizmoDriver (появление у выбранного), GizmoDragSession + стратегии осей (перемещение/вращение/масштаб); контекстные маски InteractionMaskBinder; редактирование чисел: PropertyPanel/InspectorPanel; кости: реакция ProxyRigRuntime на выбор, BoneEditMode (видимость костей, выбор кости).
- Листинги: фрагмент XRPromeonInteractable.cs (семантика tap/hold); фрагмент одной стратегии гизмо (например RingRotateStrategy); SelectionManager.cs (целиком, 22); BoneEditMode.cs (целиком, 50). Полные: XRPromeonInteractable, GizmoDriver, GizmoDragSession → Приложение Б.
- Рисунки: СКРИНШОТ гизмо на объекте (по желанию Макса – «использование гизмо»); СХЕМА состояний взаимодействия (tap/hold/grip → действия).

## 3.2.7 Создание и воспроизведение анимации
- Содержание: модель авторинга: контейнер действия выбранного объекта (данные – 3.1.2), AnimationAuthoring – операции над ключами; конвертация в кривые: AnimationClipBaker (трек → AnimationClip, тангенты Linear/Stepped); транспорт: AnimationClock (скраб, play/pause, частота кадров, остановка в конце); семплирование: AnimationPlaybackSampler – дробная позиция при воспроизведении (плавность между кадрами), целочисленный кадр при скрабе, фоновые циклические контейнеры (Loop – несколько объектов одновременно); таймлайн AnimatorPanel + view-модули (кратко, по ролям).
- Листинги: AnimationClipBaker.cs (целиком, 66 – на грани, можно фрагмент тангентов); фрагмент AnimationPlaybackSampler.cs (дробное семплирование + loop); фрагмент AnimationAuthoring.cs (постановка ключа). Полные: AnimationAuthoring, AnimationPlaybackSampler, AnimationClock, AnimatorPanel → Приложение Б.
- Рисунки: СКРИНШОТ таймлайна с ключами; СХЕМА потока «ключи → клип → семплирование → трансформации».

## 3.2.8 Экспорт данных
- Содержание: назначение (передача результата во внешние инструменты); запуск по событию из ExportPanel; снапшот состояния через SceneContext; чистая функция BuildBundle – состав пакета: scene.json (внешняя схема SceneBundle, одностороння – не для обратного импорта), копии моделей models/ и текстур textures/ с дедупликацией; запись ZIP в Documents/{productName} в фоновом потоке; ограничение встроенных ассетов (нет исходника → пометка geometryMissing) – честно, без сгущения.
- Листинги: фрагмент BuildBundle из SceneExporter.cs; SceneBundle.cs (целиком, 59 – на грани). Полный SceneExporter → Приложение Б.
- Рисунки: СКРИНШОТ панели экспорта; СХЕМА состава ZIP-пакета.
