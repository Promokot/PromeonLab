# План пункта 3.2.6 «Взаимодействие с объектами и костями»

Статус: ЧЕРНОВИК v2 НА УТВЕРЖДЕНИЕ (2026-06-05), волны 1–2 решений внесены.
База: plan-3.2.md (состав), код Assets/_App/Scripts/VrInteraction/ (+Gizmo/, Strategies/), SceneComposition/SelectionManager.cs, RigBuilder/BoneEditMode.cs, ProxyRigRuntime.cs, SpatialUi/Panels/{InspectorPanel, OutlinerPanel}.cs.
Нумерация: ПЛЕЙСХОЛДЕРЫ 3.В-N / Б.В-N (решение 1) – реальные номера при перенумерации после написания 3.2.5/3.2.7 (как 3.Э в 3.2.8). В numbering-registry до перенумерации НЕ вносится.
Рисунки: БЕЗ СХЕМ (решение Макса, 2026-06-05) – схема состояний взаимодействия из plan-3.2 ОТМЕНЕНА; только три скриншота-примера (решение 2).

## Проверка по коду (сводка фактов)

- Селекция: SelectionManager (22 строки) – единственное состояние выбора (single-select), Select(null) = сброс, смена публикует SelectionChangedEvent. Подписчики (по коду): GizmoDriver, ProxyRigRuntime, InspectorPanel, OutlinerPanel, InteractionMaskBinder, SelectionVisualSync.
- Пути выбора объекта ДВА: луч в сцене (tap по объекту) и клик по строке обозревателя иерархии (OutlinerPanel.Bind → Select(nodeId)) – оба сходятся в SelectionManager (решение 7). Обозреватель уже описан в 3.2.3 (Б.13) – здесь только механика выбора, без повтора панели.
- Визуальная реакция: SelectionVisualSync (29) по событию обходит ноды графа и переключает Selectable.SetVisualState; обводка – сторонний QuickOutline (компонент Outline), цвета/материалы – из OutlineConfig; приоритеты слоёв обводки: объект 0, кости 1, гизмо 2. Подсветка строки обозревателя – тем же событием (ApplyHighlight).
- Сброс кликом в пустоту: EmptySpaceClickDeselector (50) – триггер мимо объектов и мимо UI → Select(null). В тексте пара фраз + ссылка на полный текст в приложении (решение 3), листинга в тексте НЕТ.
- Прямое взаимодействие: XRPromeonInteractable (262) – наследник XRBaseInteractable, штатный select-flow XRI ОТКЛЮЧЁН (IsSelectableBy → false), входы контроллера читаются напрямую: триггер (activateInput) и грип (selectInput). Машина состояний Idle / TriggerPressed / TriggerRotate / GripMove: короткое нажатие триггера (окно 0,5 с) = выбор объекта; удержание триггера = вращение; удержание грипа = перемещение. ВАЖНО: вращение и перемещение доступны только если объект УЖЕ выбран (IsObjectSelected) – сначала tap-выбор, потом манипуляция. Primary-hit: при луче сквозь несколько объектов вход обрабатывает только владелец текущего хита (IsPrimaryFor). Смещения захвата – в локальном пространстве attach-трансформа руки.
- Деталь RefreshColliderRegistration (поздние коллайдеры после регистрации в XRInteractionManager) – открытый вопрос: пара фраз или опустить.
- Контекстные маски: InteractionMaskBinder (107) – живёт на корне XR-рига (персистентный, корневая шина), переключает маски кастеров обеих рук по контексту: панель гизмо открыта + есть выбор → GizmoHandles; кости видимы → BoneProxies; иначе → SceneObjects. UI-слой в маске ВСЕГДА (uGUI делит маску кастера). На ModeChangedEvent контекст сбрасывается (внутрисценные издатели «выключенное» состояние заново не публикуют).
- Гизмо (СОКРАЩЁННАЯ подача – решение 6): GizmoToolsPanel (словами) – открытие панели и выбор режима Move/Rotate/Scale; GizmoDriver (183) – видимость = панель открыта И есть цель выбора, экземпляр из префаба (GizmoConfig), следует за целью в LateUpdate, фиксированный размер из конфигурации (на кости – вдвое меньше, BoneSceneNodeMarker). GizmoDragSession (167) – СЛОВАМИ: один драг, гизмо – первоисточник (стратегия мутирует экземпляр гизмо), цель подтягивается по типу стратегии (позиция / поворот / пропорциональный фактор масштаба), режим заморожен на время драга, отмена восстанавливает исходную позу; полный текст – в приложении. Стратегии осей (AxisMove 40 / AxisScale 38 / UniformScale 33 / RingRotate 46, общий слайдер с мёртвой зоной) – СЛОВАМИ одним абзацем со ссылкой на полный текст представителя (AxisMoveStrategy) в приложении (решение 6). Bounds-fit заморожен (закомментирован) – не описываем. GizmoHandle/GizmoHierarchy/GizmoHighlightPainter – словами.
- Панель свойств – ФАКТ-ЧЕК (решение 5): PropertyPanel – ЛЕГАСИ, скрипт не присутствует ни в одном префабе и ни в одной сцене (поиск по GUID), инжект-поиск в областях возвращает null. Из текста ИСКЛЮЧЁН полностью. Отображение свойств – только InspectorPanel (283, в SceneInspectorModule.prefab): три состояния по выбору (пусто / объект / кость – по префиксу bone: идентификатора, связка с 3.2.5), отображение трансформаций (текстовые поля – «числовое редактирование» из оглавления кодом НЕ подтверждается, не заявлять), редактирование имени ноды (NodeRenamedEvent + SceneModifiedEvent), удаление ноды, тумблер Show Bones.
- Кости: BoneEditMode (50) – внутрисценный владелец «режима костей»: вход (тумблер инспектора) → SetBonesInteractive(true) + сброс выбора + BonesVisibilityChangedEvent; выход → возврат выбора ригу. SetBonesInteractive уже показан в 3.2.5 (Листинг 3.47) – НЕ повторять, отсылка.
- БЛОКИРОВКА в режиме костей (решение 8, подтверждено кодом): пока режим костей активен, выбор других объектов временно заблокирован ДВУМЯ путями: луч видит только слой BoneProxies (маска InteractionMaskBinder – объекты сцены лучу недоступны), а обозреватель игнорирует клики по строкам (OutlinerPanel: обработчик строки выходит при AnyBonesModeActive). Выход из режима – только тумблер Show Bones. Строка рига в обозревателе получает визуальный признак режима костей (SetBonesMode); сброс признака на границе сцены.
- Подсветка выбранной кости: ProxyRigRuntime.ApplyBoneSelection – цвет обводки + смена материала ромба (BoneSelectedMaterial) у выбранной, по SelectionChangedEvent. Полный ProxyRigRuntime уже в Б.21 (план 3.2.5) – в Б повторно не включается.
- ⚠ Без undo, без «ограничителей», закомментированный код не описываются.
