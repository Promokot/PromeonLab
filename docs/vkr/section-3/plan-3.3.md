## Состав текста (5–6 абзацев)

1. Открывающий: назначение технологического обеспечения; ключевое свойство – автономность (приложение исполняется на самой гарнитуре, ПК при эксплуатации не требуется, сетевое подключение не используется).
2. Целевая платформа: Meta Quest 3 – основное устройство, заявлена совместимость с Quest 2 (автономная гарнитура + пара контроллеров). Благодаря OpenXR потенциально работоспособно на иных гарнитурах, проверка не выполнялась – формулируется осторожно.
3. Стек технологий: Unity 6000.3.7f1 + URP (мобильный GPU), OpenXR как кроссплатформенный стандарт (краткое обоснование – не привязан к SDK конкретного производителя), XR Interaction Toolkit + Input System. Рисунок 3.11 – схема стека (слои: гарнитура → OpenXR-рантайм → Unity/URP/XRI → приложение; Android в схеме одним слоем, без акцента).
4. Сборка и развёртывание – кратко, один абзац: сборка под Android (IL2CPP, ARM64, Vulkan), результат – APK для установки на гарнитуру; разработка и проверка – ПК с Unity Editor и подключённой гарнитурой.
5. Условия эксплуатации – качественно, без цифр: гарнитура с парой контроллеров, свободное место в локальном хранилище под импортируемые модели/сцены/экспортные архивы, свободное пространство вокруг пользователя. Отсылка назад к 3.1.1 (все данные – в локальном хранилище устройства).
6. Ограничения текущей версии (нейтрально): функциональная проверка проводилась в среде разработки с гарнитурой Quest 3; автономная сборка полной верификации на устройстве не проходила – возможна нестабильность, в том числе в части консистентности данных; кроссплатформенная совместимость экспериментально не подтверждена. Ограничение версии и направление дальнейшей работы.
