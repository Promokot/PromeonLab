## Решения Макса (2026-06-05)

1. Содержание – «железо и система»: целевая платформа, стек, сборка, требования, ограничения. Технологический процесс обработки информации НЕ добавляем.
2. Подпункты – не более двух и только при разрастании; по умолчанию сплошной текст.
3. Требования к устройству – таблицей (Таблица 3.3). Другие OpenXR-гарнитуры потенциально совместимы, но физически не проверялись – оговорка в тексте, не в таблице.
4. Тестирование: APK на шлем НЕ устанавливался и вряд ли будет; проверка велась на ПК в Unity Editor с подключённой гарнитурой Quest 3. Это уточняет «честный абзац» ограничений.
5. Рисунки: схема стека технологий (генерирует Клод); второй рисунок – на усмотрение Клода по ходу написания (кандидат: схема сборки и развёртывания).
6. Параметры сборки – точные, из файлов репозитория (сверены, см. ниже).

## Факты из репозитория (сверено 2026-06-05)

- ProjectSettings.asset: scriptingBackend Android = IL2CPP; AndroidTargetArchitectures = ARM64; minSdkVersion = 32 (Android 12L); targetSdk – автоматический; графические API Android: Vulkan, резервный OpenGL ES 3; productName PromeonLab, версия 0.4.2.
- XR: на Android и Standalone активен единственный загрузчик OpenXR (Assets/XR/XRGeneralSettingsPerBuildTarget.asset); OculusLoader в списке провайдеров не используется.
- Пакеты (Packages/manifest.json): Unity 6000.3.7f1, URP 17.3.0, OpenXR Plugin 1.16.1, Unity OpenXR Meta 2.5.0, XR Interaction Toolkit 3.0.7, Input System 1.18.0, glTFast 6.19.0, VContainer 1.18.0.

## Состав текста (порядок абзацев)

1. Открывающий абзац: назначение технологического обеспечения – аппаратная платформа, системные средства и порядок развёртывания; ключевое свойство – автономность (приложение исполняется на самой гарнитуре, ПК при эксплуатации не требуется).
2. Целевая платформа: Meta Quest 3 – основное устройство; заявлена совместимость с Quest 2 (один класс устройств: автономная Android-гарнитура, пара контроллеров с 6DoF-трекингом). Оговорка: благодаря OpenXR потенциально работоспособно и на иных гарнитурах, проверка не выполнялась.
3. Стек технологий: Unity 6000.3.7f1 + URP 17.3.0 (мобильный GPU), OpenXR 1.16.1 как кроссплатформенный XR-стандарт (обоснование выбора против привязки к Meta SDK), XR Interaction Toolkit 3.0.7, Input System; glTFast – рантайм-импорт (отсылка к 3.2.4 без номера, если 3.2.4 ещё не принят – формулировка нейтральная). Рисунок 3.11 – схема стека (слои: гарнитура → Android → OpenXR-рантайм → Unity/URP/XRI → приложение).
4. Сборка и развёртывание: цель Android, IL2CPP + ARM64, минимальная версия Android 12L (API 32), Vulkan с резервом OpenGL ES 3; результат – APK, устанавливаемый на гарнитуру. Возможен Рисунок 3.12 – схема «проект → сборка APK → устройство» (решу при написании, по объёму).
5. Среда разработки и проверки: ПК с Unity Editor; функциональная проверка – гарнитура Quest 3, подключённая к ПК (исполнение в редакторе, OpenXR-рантайм ПК). Кратко, 1 абзац – стыкуется с абзацем ограничений.
6. Данные на устройстве: локальное хранилище Application.persistentDataPath (отсылка назад к 3.1.1), экспортные архивы в Documents, импорт пользовательских файлов через встроенный файловый обозреватель без выхода из VR. Интернет-соединение не требуется.
7. Таблица 3.3 «Требования к устройству»: гарнитура (Quest 3; совместимость с Quest 2), контроллеры (пара), свободное место (приложение + данные пользователя), свободное пространство вокруг пользователя, подключение к сети (не требуется).
8. Ограничения текущей версии (нейтрально, без самокритики): функциональная проверка проводилась в среде разработки с гарнитурой Quest 3; автономная Android-сборка полной верификации на устройстве не проходила – возможна нестабильность, в том числе в части консистентности данных. Формулируется как ограничение версии и направление дальнейшей работы.
