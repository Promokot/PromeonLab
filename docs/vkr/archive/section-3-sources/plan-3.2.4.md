## Проверка по коду (сводка фактов)

- Три библиотеки за общим интерфейсом IAssetLibrary, записи – ILabAsset:
  - BuiltinAssetLibrary – ScriptableObject, только чтение (Add/Remove бросают исключение); записи BuiltinLabAsset с префабом, иконкой-спрайтом и шаблоном, испечённым в редакторе;
  - ImportedAssetLibrary – imported-lib.json (схема данных v2: шаблон в каждой записи), загрузка при старте корневой области (IStartable);
  - SavedAssetLibrary – saved-lib.json; формат и хранение реализованы, но поток наполнения (сохранение ассета из сцены) не реализован – по правилу 6.3 одно честное предложение, без подробностей.
- AssetRegistry – поиск записи по ссылке AssetRef (выбор библиотеки по полю Source, перебор записей по Id). 28 строк, можно целиком.
- Конвейер импорта (ImportPipeline, корневая область, IStartable):
  FileBrowserPanel (обёртка стороннего SimpleFileBrowser, уже упомянута в 3.2.2) → FilePickedEvent → выбор импортёра по расширению (GltfAssetImporter: .glb/.gltf через glTFast; ImageAssetImporter: .png/.jpg/.jpeg) → ImportRequestedEvent → ImportWizardPanel (имя, тип Object/Rig/Reference, для ригов – ось концевых костей с инверсией) → ImportConfirmedEvent → RunImportAsync: копирование источника (ImportedSourceProvider → sources/{id}.{ext}, ссылка относительная), построение шаблона восстановления (build-once, BuildAsync по типу), доустановка осей костей из мастера, миниатюра, запись в библиотеку + сохранение, AssetImportedEvent.
- Миниатюры (ThumbnailRenderer): офскрин-камера, модель грузится во временную точку далеко под сценой, кадрирование по ограничивающей сфере (FrameDistance), PNG 256x256 в thumbnails/{id}.png; для изображений миниатюрой служит сам файл-источник (без рендера); сбой миниатюры импорт не прерывает.
- Браузер (AssetBrowserPanel, постоянная панель регионной модели): вкладки трёх библиотек, сетка карточек LabAsset_Item (элемент списка: миниатюра + имя, событие выбора), ResolveIcon (спрайт Builtin или загрузка PNG по ThumbnailRef с кешем, null тоже кешируется), кнопки: добавить (открывает файловый браузер, флаг возврата региона), разместить (активна только в режимах редактирования/песочницы), удалить (только не-Builtin: запись + файл-источник).
- Размещение: кнопка вычисляет точку перед камерой и разворот лицом к пользователю → AssetSpawnRequestedEvent → AssetSpawner (внутрисценная область): смещение spawnOffset из шаблона (подъём референсных изображений над полом), RestoreAsync реестра билдеров, регистрация ноды в графе (AddNode), внедрение зависимостей в иерархию.
- AssetEntityBuilderRegistry.RestoreAsync – единая точка восстановления (и для спавна, и для загрузки сцены из 3.2.3): диспетчеризация по типу ассета, отказ для Builtin без шаблона, применение InteractionCapability.Apply из шаблона (слой, коллайдер, выбираемость), регистрация селекторных коллайдеров для ригов (вперёд, словесно).
- BuiltinRecipeBaker (editor-only, папка Editor/): измерение префаба в изолированной превью-сцене тем же синхронным ядром, что и рантайм-импорт; запись приватных полей через рефлексию; для Reference генерируется и префаб.
- Связка с 3.1.2: шаблон восстановления AssetEntityRecipe (Листинг 3.3) – момент «build» создаёт шаблон, «restore» применяет. Обратная отсылка – одна, структурно значимая.
- Отсылка вперёд для скелетных моделей – словесная, без номера пункта (правило 5.9).

## Предлагаемая структура текста (порядок абзацев)

1. Вводный абзац: ассеты – строительный материал сцен; деление на три библиотеки по происхождению; общий интерфейс и записи; AssetRegistry как точка поиска по ссылке AssetRef (узел к загрузке сцены из 3.2.3).
2. Три библиотеки: Builtin (ScriptableObject, только чтение, испечённые шаблоны), Imported (json, схема v2), Saved (честная оговорка о нереализованном потоке). [Кандидат на Таблицу 3.3]
3. Конвейер импорта, часть 1: файловый браузер → FilePickedEvent → выбор импортёра по расширению → запрос мастеру. [Листинг: фрагмент ImportPipeline]
4. Мастер импорта: поля, типы, оси костей; Confirmed=false при отмене. [СКРИНШОТ мастера]
5. Конвейер импорта, часть 2: RunImportAsync – копирование источника, build-once, миниатюра, запись, AssetImportedEvent. [Листинг: фрагмент ImportPipeline; СХЕМА конвейера]
6. Миниатюры: ThumbnailRenderer, офскрин-рендер, кадрирование; изображения без рендера. [Листинг: фрагмент]
7. Браузер ассетов: вкладки, сетка карточек, ResolveIcon с кешем, кнопки. [СКРИНШОТ браузера]
8. Размещение: точка перед камерой → AssetSpawnRequestedEvent → AssetSpawner → RestoreAsync + InteractionCapability; restore-many и для загрузки сцены. [Листинги: AssetSpawner целиком, фрагмент RestoreAsync]
9. Бейк встроенных ассетов: BuiltinRecipeBaker, явная пометка «инструмент редактора, в сборку не входит». [Листинг: фрагмент]
10. Финал-переход: импортированный риг получает скелетную структуру – тема следующего пункта (словесно). [ПАРНЫЙ РИСУНОК Blender ↔ приложение]
