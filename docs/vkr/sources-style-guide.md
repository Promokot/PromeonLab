# Инструкция: оформление записи в списке использованных источников

Основание: методичка «Требования к оформлению ВКР» (раздел 9, Приложение Г) + решения
нормоконтроля и Макса (REQUIREMENTS.md, п. 1.1, 4.x). Эталон применения: sources-final.md.

## 1. Общие правила списка

1. Список разбит по разделам работы. Заголовки групп: «1. АНАЛИТИЧЕСКИЙ РАЗДЕЛ»,
   «2. СПЕЦИАЛЬНЫЙ РАЗДЕЛ», «3. ТЕХНОЛОГИЧЕСКИЙ РАЗДЕЛ». Группы «ВВЕДЕНИЕ» нет –
   источники введения идут в раздел 1.
2. Номер источника: <номер раздела>.<порядковый номер в разделе>, с точкой после: «1.5.», «3.12.».
3. Внутри раздела источники идут В ПОРЯДКЕ ПЕРВОГО УПОМИНАНИЯ в тексте записки.
4. Ссылка в тексте: номер в квадратных скобках – [1.5]; несколько – [1.5, 1.6]. Точка
   ставится ПОСЛЕ ссылки: «...текст [1.5].» Страницы НЕ указываем (решение Макса).
5. Требования к составу: всего не менее 40 источников; год издания не ранее 2021.
6. Запрещены: Википедия, YouTube, соцсети, КиберЛенинка, ресурсы Meta Platforms.
   URL чистим от utm-меток и srsltid.

## 2. Знаки и их роль (что за что отвечает)

| Знак | Роль | Пример |
|---|---|---|
| «–» (en dash, с пробелами) | Разделитель ОБЛАСТЕЙ описания: издание, город, год, объём | «. – 2-е изд. – СПб.:Питер, 2023. – 448 с.» |
| «—» (em dash) | НЕ ИСПОЛЬЗУЕТСЯ. В методичке стоит «—», но по решению нормоконтроля везде заменяется на «–» | |
| «//» | Отделяет название МЕНЬШЕГО от БОЛЬШЕГО: статья // журнал, страница // сайт | «Кривые... // Хабр» |
| «/» | Сведения об ответственности после названия: переводчик, редактор | «/ пер. с англ. Р.Н.Рагимова» |
| «:» после города | Перед издательством, БЕЗ пробела после «М.:», «СПб.:» (п. 9.9) | «М.:Юрайт» |
| «:» внутри названия | Подзаголовок – как в оригинале | «Язык С#. Базовый курс: учебное пособие» |
| «,» перед годом | Отделяет издательство от года | «Питер, 2023» |
| Точка в конце записи | Обязательна всегда | |

Инициалы: после фамилии, без пробелов между собой: «Боресков А.В.» (не «А. В.»).
Несколько авторов перечисляются через запятую, все в начале записи.
Города-сокращения: Москва – «М.», Санкт-Петербург – «СПб.», Ленинград – «Л.»;
прочие города пишутся полностью (Boston, New York).

## 3. Шаблоны по типам источников

### 3.1. Книга (печатное издание)

    Фамилия И.О.[, Фамилия И.О.] Название[: подзаголовок][ / сведения об ответственности]. – [N-е изд. – ]Город:Издательство, год. – N с.

Порядок областей: автор(ы) → название → (перевод/редактор) → издание → город:издательство, год → объём.

Примеры из принятого списка:
- Хокинг Дж. Unity в действии. Мультиплатформенная разработка на C#. – 3-е изд. – СПб.:Питер, 2023. – 448 с.
- Линовес Дж. Виртуальная реальность в Unity / пер. с англ. Р.Н.Рагимова. – М.:ДМК Пресс, 2023. – 316 с.
- LaViola J.J., Kruijff E., McMahan R.P., Bowman D.A., Poupyrev I. 3D User Interfaces: Theory and Practice. – 2nd ed. – Boston: Addison-Wesley, 2017. – 624 p.

Для иностранных книг объём пишется «p.» вместо «с.», город без сокращения.

### 3.2. Статья в журнале

    Фамилия И.О.[, ...] Название статьи // Название журнала. – Год. – Vol. X[, № Y]. – С. A–B | Article N.

Порядок: автор(ы) → название статьи → «//» → журнал → год → том/номер → страницы или номер статьи.
Русский журнал: «Т. 9. – Вып. 1. – С. 157–159»; иностранный: «Vol. 272. – Article 170274» или «P. 59–76».

Примеры:
- Hu Z., Liu L. Research on the application of virtual reality technology in 3D animation creation // Optik. – 2023. – Vol. 272. – Article 170274.
- Boletsis C., Chasanidou D. A Typology of Virtual Reality Locomotion Techniques // Multimodal Technologies and Interaction. – 2022. – Vol. 6, № 9. – Article 72.

### 3.3. Доклад в сборнике конференции

    Фамилия И.О.[, ...] Название доклада // Название конференции. – Год. – С. A–B.

Пример:
- Bergström J., Dalsgaard T.-S., Alexander J., Hornbæk K. How to Evaluate Object Selection and Manipulation in VR? Guidelines from 20 Years of Studies // Proceedings of the 2021 CHI Conference on Human Factors in Computing Systems. – 2021.

### 3.4. Web-ресурс (электронный источник)

    Название страницы/материала // Название сайта: – URL: https://... (дата обращения: ДД.ММ.ГГГГ). – Текст электронный.

Порядок: название материала → «//» → сайт → «: – URL:» → адрес → дата обращения в скобках → пометка «Текст электронный».
- Автор указывается в начале, только если он явно назван (Nielsen J. ...).
- Год публикации в web-записи НЕ пишется – его роль выполняет дата обращения.
- URL без закрывающего слэша не дописываем/не обрезаем – как в адресной строке, но без utm/srsltid.

Примеры:
- JsonUtility // Unity Scripting API: – URL: https://docs.unity3d.com/ScriptReference/JsonUtility.html (дата обращения: 04.06.2026). – Текст электронный.
- EventBus – Система событий для Unity // Хабр: – URL: https://habr.com/ru/articles/527418/ (дата обращения: 04.06.2026). – Текст электронный.

## 4. Алгоритм построения записи (пошагово)

1. Определи ТИП источника: книга / статья журнала / доклад конференции / web-страница.
2. Определи РАЗДЕЛ работы, где источник упоминается впервые, – это первая цифра номера.
3. Собери элементы по шаблону своего типа (п. 3.x), строго в указанном порядке.
4. Расставь разделители: «–» между областями, «//» перед источником-носителем,
   «:» без пробела после «М.»/«СПб.», запятая перед годом, точка в конце.
5. Проверь анти-правила: нет «—», нет пробелов в инициалах, нет utm в URL, год ≥ 2021
   (для книг/статей), источник не из запрещённого списка.
6. Поставь номер <раздел>.<порядок> по месту первого упоминания и точку после номера.
7. После любых перестановок текста – пересортируй и перенумеруй раздел заново.

## 5. Чек-лист самопроверки (быстрый)

- [ ] номер вида X.Y. и группа соответствует разделу первого упоминания;
- [ ] тип записи определён верно, ни один элемент шаблона не пропущен;
- [ ] все тире – «–» с пробелами; точка в конце записи;
- [ ] «М.:Издательство» / «СПб.:Издательство» – без пробела; инициалы слитно;
- [ ] web: есть «// Сайт: – URL:», дата обращения, «– Текст электронный.»;
- [ ] книга/статья: год ≥ 2021 (исключения – только осознанные решения);
- [ ] URL открывается и ведёт на содержательную страницу (не «микро-страницу»);
- [ ] источник не из запрещённых: Википедия, YouTube, соцсети, КиберЛенинка, Meta.
