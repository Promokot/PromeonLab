# Требования к тексту ВКР (рабочий перечень)

Статус: v2 (2026-06-04). Дополняется по мере появления новых правил.
Источники: методичка (notes/formatting-requirements-pdf.md), ответы нормоконтроля, правила Макса.

## 1. Анти-ИИ-маркеры (приоритет)

1.1. Длинное (тройное) тире «—» НЕ используется – по ответу нормоконтроля (Дарья Александровна) заменяется на двойное «–» (en dash) с пробелами. Это касается всего: прозы, подписей рисунков/таблиц/листингов, списка источников. В Word финальная замена делается через поиск-замену.
1.2. Никаких следов генерации в URL источников: вычищать utm_source=chatgpt.com и подобные параметры (в v11 такие есть – исправить).
1.3. Избегать шаблонных ИИ-оборотов: «стоит отметить», «важно подчеркнуть», «в современном мире», «играет ключевую роль», «является неотъемлемой частью», навязчивых «таким образом» в начале абзацев.
1.4. Не злоупотреблять маркированными списками в прозе – связный текст предпочтительнее перечислений, списки только там, где перечисление по сути (методы класса, критерии).
1.5. Разнообразить длину предложений; не строить все абзацы по одной схеме «тезис, расшифровка, вывод».
1.6. Текст эмоционально нейтральный: без оценочных эпитетов («удобный», «элегантный», «мощный»), без восклицаний и рекламных формулировок.

## 2. Пунктуация и типографика (методичка)

2.1. Дефис «-» – сложные слова и диапазоны (100-150 см), без пробелов.
2.2. «–» – знак вычитания в формулах; по решению нормоконтроля также выполняет роль тире в тексте (см. 1.1).
2.3. Кавычки русские «…»; в английских фрагментах "…".
2.4. Пробел после знака препинания, не перед; нет пробела после открывающей скобки/кавычки.
2.5. Математические знаки в тексте – словами («плюс», «равно», «больше»); знаками – только в формулах.
2.6. Числа арабскими, без падежных окончаний; единица измерения один раз после последнего числа ряда.
2.7. Без переносов слов, без сносок, без нерасшифрованных сокращений.

## 3. Структурные элементы

3.1. Рисунок: после абзаца первого упоминания (или след. страница), по центру. Подпись ПОД рисунком: «Рисунок 3.N – Название» (без точки в конце; слово не сокращать). Нумерация в пределах раздела.
3.2. Таблица: название НАД таблицей «Таблица 3.N – Название»; перенос – «Продолжение Таблицы 3.N»; без графы «№ п/п».
3.3. Листинг: надпись ПЕРЕД листингом «Листинг 3.N – Название»; фрагменты в тексте, полный код в приложении; ссылки на строки с шагом 10: «Листинг 3.2 {30, 50}».
3.4. Формулы: по центру, пустые строки до/после, номер (3.N) по правому краю; нумеровать только те, на которые есть ссылки; пояснения через «где» без двоеточия.
3.5. Ссылки на элементы с большой буквы: «(Рисунок 3.2)», «в соответствии с Таблицей 3.1», «(Листинг 3.4)», «(Приложение Б)».
3.6. Списки: один маркер на всю работу; маркированные – со строчной, «;», последний «.»; нумерованные – с прописной, «.».
3.7. У каждого раздела – подраздел «Выводы» в конце.

## 4. Источники

4.1. Всего ≥ 40, год не ранее 2021. Текущий список v11 признан непригодным: после написания текстов большая часть источников подбирается заново.
4.2. Список по разделам (1 / 2 / 3); внутри – в порядке упоминания. Группы «ВВЕДЕНИЕ» нет: источники введения переносятся в раздел 1 (решение Макса, 2026-06-04).
4.3. Ссылки в тексте: [1.5], [3.2]; точка после ссылки. Страницы в ссылках НЕ указываем (решение Макса: выше шанс попадания в источник при проверке).
4.4. Номер источника: <раздел>.<порядок> (в v11 сквозная нумерация – переделать).
4.5. Оформление записей: книги/журналы/web по образцу методички; для web – «// Сайт: – URL: … (дата обращения: ДД.ММ.ГГГГ). – Текст электронный.»; города М., СПб.; без голых URL (в v11 есть – переделать).

## 5. Стиль изложения

5.1. Безличный научно-академический стиль: «реализовано», «используется», «применяется» (без «мы», «я»), без эмоциональной окраски (требование Макса; заменяет ОбщиеРекомендации.txt).
5.2. Имена классов/файлов/методов – латиницей без выделения, как обычные слова (стиль v11): PathProvider, scene.json.
5.3. Заголовки: без точки в конце; нумерованные разделы – только содержательные главы.
5.4. Терминология едина с главами 1–2 (нода, риг, ключевой кадр, гизмо и т.д.).
5.5. Уровни структуры называть строго по методичке: раздел (3) → подраздел (3.1) → пункт (3.1.1); не путать в самоссылках.
5.6. Термин «подсистема» в разделе 3 минимизировать (чтобы не создавать ложную привязку к подсистемам главы 2); имена модулей (StorageCore и т.п.) употреблять как собственные имена.
5.7. Внутри подраздела держать единый предмет изложения (в 3.1 – данные, не функционал) и связки-переходы между пунктами; не перескакивать между темами без перехода.

## 6. Технические правила процесса

6.1. В .md-файлах раздел 3 пишем сразу с правильной нумерацией рисунков/таблиц/листингов в пределах раздела.
6.2. Факты о реализации – только из актуального кода/доков (review-2026-06-03, CLAUDE.md); запрещено переносить из старого раздела 3 без проверки (см. notes/v11-thesis-notes.md, список устаревшего).
6.3. Нереализованные фичи (FBX-экспорт, IK-солвер, NLA, undo) не описывать как готовые; допустимо упоминать как направления развития в заключении.
6.4. Скрипты из _Archive не цитировать и не включать в приложения.
