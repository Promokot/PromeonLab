## Структура (фактическая)

- ВВЕДЕНИЕ — актуальность, новизна, практическая значимость, цель + 5 задач. Ссылки вида [1.1, 1.2].
- 1 АНАЛИТИЧЕСКИЙ РАЗДЕЛ (стр. 7–26): 1.1 обзор Quill/Tvori/AnimVR + сравнительный анализ; 1.2 методы анимации (таймлайн, объектная, риггинг, интерполяция); 1.3 трекинг; 1.4 функциональные требования + выбор стека; Выводы.
- 2 СПЕЦИАЛЬНЫЙ РАЗДЕЛ (стр. 27–41): 2.1 функциональная схема; 2.2–2.6 подсистемы (файловые ресурсы, сцены, скелетные структуры, анимация, экспорт); Выводы. Уровень — проектный (схемы, без кода).
- 3 ТЕХНОЛОГИЧЕСКИЙ РАЗДЕЛ (стр. 42–65): 3.1 Информационное (3.1.1 StorageCore, 3.1.2 структура данных сцены, 3.1.3 сериализация, 3.1.4 автосейв/dirty); 3.2 Программное (3.2.1 главное меню, 3.2.2 управление сценами, 3.2.3 добавление ассетов, 3.2.4 взаимодействие с объектами, 3.2.5 риг, 3.2.6 анимации); 3.3 Технологическое (½ страницы: требования к железу); Выводы.
- ЗАКЛЮЧЕНИЕ, СПИСОК ИСТОЧНИКОВ (стр. 69–71), ПРИЛОЖЕНИЯ: А — графический интерфейс (рисунки), Б — листинг кода.

## Стиль раздела 3 (берём как образец подачи)

- Короткая проза → листинг фрагмента (Листинг 3.N) или рисунок-скриншот → пояснение.
- Перечни методов класса — нумерованным списком с однострочными пояснениями.
- Полные скрипты выносятся в Приложение Б («…представленном в Листинге Б.1»).
- Ссылки на источники в тексте раздела 3 редки ([3.1] один раз).
- 3.2 опирается на скриншоты UI (большинство из 23 рисунков).
