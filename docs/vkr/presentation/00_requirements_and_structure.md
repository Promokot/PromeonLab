# Презентация к защите ВКР — требования и структура

Источник: «Шаблон презентации к защите.pptx» (кафедра КБ-9, профиль «Технологии визуального
программирования»), построенный на брендбуке РТУ МИРЭА (утверждён с февраля 2026).

> **Про исходники.** Брендбук (`Brandbook-RTU-MIREA.pdf`) и архивы шаблонов
> (`SHablon-dlya-prezentatsii-RTU-MIREA_Arial.rar`, `..._Osnovnoy-shrift.rar`) — бинарные файлы;
> доступными мне веб-инструментами их не скачать. Но **сам шаблон из инбокса уже сделан по
> брендбуку**, поэтому фирменный стиль (палитра, шрифт, декоративные элементы) я вытащил прямо
> из него — см. `resources/brand-template-extracted/`. Ссылки на оригиналы — внизу файла, чтобы
> ты докачал их вручную, когда провайдер позволит.

## 1. Требования к оформлению (из брендбука, извлечено из темы шаблона)

> **⚠️ Уточнение по брендбуку (полный брендбук прочитан 2026-06-09).** Все 266 страниц `Brandbook-RTU-MIREA.pdf` разобраны; авторитетная выжимка по оформлению — **`Шпаргалка_оформление_РТУ_МИРЭА.md`** в этой же папке. Таблицы ниже извлекались из темы шаблона ДО прочтения брендбука, поэтому уточняются так:
> - **Основной цвет — тёмно-синий `#355CA8`** (не фиолетовый). Фиолетовый `#3D378D` — цвет Передовой инженерной школы СВЧ-электроники, к КБ-9 не относится. Акценты КБ-9 — бирюза Института кибербезопасности `#006A81` (доп. `#004352`, `#94CCD9`); общая гамма — `#5FC1CF`, `#A7D9DE`, серый `#E6E6E6`, градиент `#355CA8`→`#3772B3`→`#5FC1CF`.
> - **Основной шрифт — Helvetica Neue** (начертания Roman/Medium/Bold). **Arial — запасной**, на случай отсутствия Helvetica (для него есть отдельный вариант шаблона `..._arial`).
> - Заголовки слайдов — **Bold тёмно-синий**, не чёрный.
> - На презентации используется **логотип** (круг «РТУ МИРЭА»), **не герб**; охранное поле = 2/5 ширины знака; размещение — правый верхний угол.

| Параметр | Значение |
|---|---|
| Формат слайда | 16:9 (12192000×6858000 EMU = 33,87×19,05 см) |
| Шрифт | **Arial** (вариант «Arial» брендового шаблона; есть и вариант на основном фирменном шрифте) |
| Размер заголовка слайда | ~18 pt |
| Размер основного текста | 12–14 pt |
| Крупный акцентный текст | 20 pt |
| Основной цвет текста | чёрный `#000000` на белом фоне |

> в финальной презентации её быть не должно.

### Декоративные элементы
В `resources/brand-template-extracted/media/` — набор фирменных градиентных «сфер»
(синие, бирюзовые, красные полукруги-градиенты). Это официальные оформительские элементы
шаблона; их можно использовать как фон/акцент по углам слайдов. Свои «левые» рамки, тени,
клипарт добавлять нельзя — только то, что даёт брендбук/шаблон.

### Правила (текстом из техслайда шаблона)
- Перечень слайдов ниже — **обязательный** для защиты на КБ-9.
- Подзаголовки слайдов **обязательны** (их надо уточнить под свою тему).
- Размеры шрифтов менять можно; компоновать содержимое внутри слайда — как угодно.
- **Нельзя нарушать правила брендбука** (палитра, шрифт, логотип).
- Все задачи из введения должны быть раскрыты по ходу презентации.
- Первый служебный слайд-инструкцию из шаблона — удалить.

## 2. Обязательная структура слайдов (каркас шаблона)

| № слайда | Назначение | Обязательность |
|---|---|---|
| 1 | Титул: тема, ФИО, руководитель, кафедра, направление, профиль | обязателен |
| 2 | Введение: цель, актуальность, новизна, практическая значимость + **список задач** | обязателен |
| 3..n | Аналитический раздел (по задачам), 2–n слайдов | обязателен |
| — | Анализ существующих решений + **таблица сравнения** | обязателен |
| — | Постановка задачи: функциональные и нефункциональные требования | обязателен |
| — | Функциональная схема / общая архитектура приложения | обязателен |
| — | Блок-схемы / диаграммы / mind-карты принципов работы (2–n) | обязателен |
| — | Визуализация работы приложения (скриншоты + код из 3-го раздела), **мин. 3 слайда**: информационное / программное / техническое обеспечение | обязателен |
| — | **Демонстрация работы — видео обзора функций (~2 мин)** | обязателен |
| — | Заключение: «цель достигнута, задачи выполнены…» + перечень решённых задач (= задачи из введения) | обязателен |

> Нумерация в заголовках слайдов привязана к номеру задачи из введения
> (напр. «1. Рассмотрение алгоритмов…»; при дроблении — 4.1, 4.2 …).

## 3. Тайминг (выступление 5 минут)

Видео-демо занимает ~2 мин → на устный доклад остаётся **~3 минуты** (≈ 380–430 слов).
Значит на каждый «говорящий» слайд — очень мало времени, нужен жёсткий отбор: 1 слайд ≈ 15–25 сек.
