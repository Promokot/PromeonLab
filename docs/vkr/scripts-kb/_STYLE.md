# _STYLE — как устроены заметки базы

> [!info] Назначение
> Внутренний стандарт оформления для всех заметок «Скрипты ВКР». Не для защиты — рабочий ориентир, чтобы все страницы выглядели одинаково и связывались между собой.

### Структура заметки скрипта

- ##### Три уровня вложенности (обязательно)
    - `###` — **заголовок раздела** (Обзор / Разбор кода / К защите / Связи).
    - `#####` — **заголовок блока** внутри раздела (метод, фрагмент, вопрос-группа).
    - содержимое блока — текст, код, цитаты-callout, списки.
- ##### Разделы заметки скрипта по порядку
    - `> [!info] Назначение` — 1–3 предложения: что делает класс, к какой подсистеме относится, номера листингов (теловой 3.x и/или Приложение Б.x).
    - `### Обзор` → блоки `##### Роль и место`, `##### Ключевые методы`.
    - `### Разбор кода` → по блоку `##### <метод/фрагмент>` на каждую важную единицу; внутри блока сначала ```csharp фрагмент реального кода```, затем `> ` callout с разбором.
    - `### К защите` → блок `##### Вероятные вопросы` с парами **В:** / **О:** в callout `> [!question]`.
    - `### Связи` → строка из `[[ссылок]]` через ` · ` (другие скрипты, события, концепты).

### Правила глубины (режим «обзор + опасные строки»)

- ##### Что разбираем построчно
    - Только **неочевидные/опасные** строки: хитрые касты, порядок операций, защиты от гонок/реентерабельности, дебаунсы, снимки коллекций, синхронные-до-async участки, магические числа, краевые случаи.
    - Очевидные строки (геттеры, тривиальные присваивания) — не комментируем построчно, упоминаем в обзоре.
- ##### Чем заземляемся
    - Каждое утверждение — из **реально процитированных строк** или из текста раздела 3 ВКР. Не выдумывать поведение, которого нет в коде. Если не уверены — пометить «(уточнить)».
    - Код цитировать дословно из исходника. Имена методов/полей — точные.

### Калибровка глубины — эталон A (EventBus.Publish)

```csharp
foreach (var handler in list.ToArray())
    ((Action<T>)handler).Invoke(message);
```

> `list.ToArray()` — не косметика, а защита. `_handlers[type]` хранит подписчиков как `List<object>`. Если обработчик внутри `Invoke` вызовет `Subscribe`/`Unsubscribe` на тот же тип (панель отписывается, получив событие), коллекция изменится во время `foreach` → `InvalidOperationException`. `ToArray()` снимает копию на момент рассылки. Каст `(Action<T>)handler` обязателен: список хранит `object` (тип стёрт, чтобы один словарь держал обработчики всех типов); `where T : struct` гарантирует не-`null` сообщение и точное сравнение типов через `typeof(T)`.

### Калибровка глубины — эталон B (AnimationClock.Tick)

```csharp
_accumulated += Time.deltaTime * Fps;
var next = Mathf.FloorToInt(_accumulated);
if (next == CurrentFrame) return;
```

> `_accumulated` — дробная позиция в кадрах. За экранный кадр прибавляется `Δt·Fps` (при 24 fps анимации и 90 fps дисплея ≈ 0,27 кадра). `CurrentFrame` (целое) меняется только при переходе через целую границу — `FloorToInt` + ранний `return`, чтобы не слать `FrameChangedEvent` 90 раз/с. Плавность даёт не это поле, а `CurrentFrameContinuous => _accumulated` (дробное), которое сэмплер читает напрямую; целочисленное событие лишь двигает ползунок. Поэтому движение не «квантуется» по 24 fps.

### Соглашения об именах заметок и ссылках

- ##### Имена файлов
    - Заметка скрипта = имя класса без `.cs`: `EventBus.md` → ссылка `[[EventBus]]`.
    - Сгруппированные структуры — осмысленное имя: `Структуры данных сцены.md`, `Структуры анимационных данных.md`.
    - Концепты: `Паттерн Publish-Subscribe.md`, `Внедрение зависимостей (VContainer).md`.
    - Навигация: `Скрипты ВКР.md` (главная), `3.1 Информационное обеспечение.md`, `3.2 Программное обеспечение.md`.
- ##### Ссылки
    - Внутри текста и в `### Связи` ставить `[[ИмяЗаметки]]`; для событий — `[[SceneModifiedEvent]]` (даже если файла-события пока нет — это валидная «висячая» ссылка-задел).

### Оформление навигационных страниц

- Ориентир — дашборд ВКР: `### Раздел` → `##### Блок` → маркированный список `[[ссылок]]`, разделители `___`, врезки `> [!info]`. Никаких таблиц на главной — только блоки со ссылками.
