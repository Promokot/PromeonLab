# Дебаунс записи

> [!info] Назначение
> Приём отложенной записи: серия частых запросов на сохранение схлопывается в одну дисковую операцию. Применён к записи `animation.json` в [[AnimationStorage]].

### Суть подхода

- ##### Проблема
    - Каждая правка ключа анимации (постановка, перенос, удаление, смена длины) логически требует записи файла. При активном редактировании это десятки операций в секунду — лишняя нагрузка на флеш-память устройства Quest.
- ##### Решение
    - Запрос на запись не пишет сразу, а ставит таймер на **200 мс**. Новый запрос за это время **перезапускает отсчёт** (отменяет прежний таймер). Файл пишется только в первую паузу — серия правок превращается в одну запись.
- ##### Цена
    - При аварийном завершении теряется не больше 200 мс работы — приемлемый компромисс. Граф сцены сохраняется иначе (по событию выхода, см. [[SceneAutoSaver]]), у анимации — именно дебаунс.

### Где применяется

- ##### [[AnimationStorage]]
    - Перезапуск отсчёта реализован отменой `CancellationTokenSource`: предыдущая отложенная задача отменяется, заводится новая. `TaskCanceledException` при отмене поглощается (это штатный путь, не ошибка). `Dispose` отменяет висящий таймер.

### К защите

> [!question] Вероятные вопросы
> **В: Почему именно 200 мс?** О: Подобрано как баланс: достаточно мало, чтобы потеря при сбое была незаметной, и достаточно велико, чтобы схлопнуть типичную серию правок в одну запись.
>
> **В: Чем дебаунс отличается от троттлинга?** О: Троттлинг пишет не чаще раза в интервал (периодически во время активности). Дебаунс ждёт **паузы** — пока правки идут, запись откладывается полностью.
>
> **В: Не потеряются ли данные при выходе из приложения?** О: В худшем случае — последние ≤200 мс. Критичные для пользователя данные графа сцены сохраняются по другому механизму (событие выхода из режима), не через дебаунс.
>
> **В: Почему граф сцены сохраняется не так?** О: Граф пишется при выходе из режима по снимку (синхронный захват состояния), потому что после начала перехода объекты сцены выгружаются — отложенная запись там не успела бы.
