# Событийные структуры (Event)

> [!info] Назначение
> Все межмодульные сообщения проекта — это `struct` с суффиксом `Event`, рассылаемые через [[EventBus]]. Здесь — как они устроены, **как конструируются** (у них нет собственных конструкторов) и почему именно `struct`.

### Анатомия события

- ##### Это всегда `public struct` + публичные поля
    - Пример целиком: `public struct FrameChangedEvent { public int Frame; }`. Никаких методов, свойств, наследования — только данные. Публичные поля здесь допустимы (это правило-исключение проекта для данных, сериализуемых/передаваемых движком и шиной, в отличие от поведенческих классов, где поля только `[SerializeField] private`).
    - Лежат в подпапке `Events/` своей подсистемы (`Animation/Events/`, `SceneComposition/Events/` …). Имя файла = имя типа.
- ##### Маркеры без полей
    - Часть событий — пустые: `public struct SceneModifiedEvent { }`, `SceneClosedEvent`, `GizmoToolsPanelOpenedEvent`. Они несут только сам факт («сцена изменилась»), данных не нужно.

### Конструирование («конструкторы»)

- ##### Своих конструкторов у событий НЕТ
    - Ни одно событие не объявляет конструктор. В C# у любой структуры есть неявный конструктор без параметров, который обнуляет все поля. Поэтому событие создаётся в два логических шага: сначала **дефолтная инициализация** всех полей, затем **инициализатор объекта** заполняет нужные.
    - Канонический вид публикации:
      ```csharp
      _bus.Publish(new FrameChangedEvent { Frame = CurrentFrame });
      ```
      `new FrameChangedEvent { … }` — это не вызов пользовательского конструктора, а синтаксис инициализатора: создаётся значение со всеми полями по умолчанию, после чего `Frame` присваивается.
- ##### Чему равны незаданные поля
    - Всё, что не указано в `{ … }`, остаётся в значении по умолчанию: `string → null`, `int → 0`, `bool → false`, **`enum → 0` (его первый член)**. Это важно: `new ImportConfirmedEvent { Confirmed = false, FilePath = _filePath }` оставит `ChosenType = AssetType.Object` и `TerminalBonesAxis = TerminalBoneAxis.Auto`, потому что это нулевые члены перечислений. Забыл поле — получил «первый» вариант, а не ошибку компиляции.
- ##### Пустое событие
    - `_bus.Publish(new SceneModifiedEvent());` — обычный вызов неявного конструктора структуры; фигурных скобок-инициализатора нет, заполнять нечего.

### Почему `struct`, а не `class`

- ##### Семантика значения и отсутствие `null`
    - Ограничение шины `where T : struct` ([[EventBus]]) гарантирует, что сообщение — значимый тип и не может быть `null`; подписчику не нужны проверки на пустую ссылку.
- ##### Дешёвая рассылка без аллокаций
    - В `Publish<T>(T message)` событие передаётся по значению и доходит до `Invoke(message)` без упаковки в кучу (боксинга подвергаются обработчики, хранимые как `object`, но не само сообщение). Для VR-приложения, где события летят каждый кадр (`FrameChangedEvent`), это снижает нагрузку на сборщик мусора.
- ##### Копия, а не общая ссылка
    - Подписчик получает копию структуры — он не может случайно изменить её и повлиять на других получателей.

### К защите

> [!question] Вероятные вопросы
> **В: Какой у событий конструктор?** О: Только неявный конструктор структуры без параметров (обнуляет поля). Своих конструкторов события не объявляют — поля заполняются инициализатором объекта `new X { Field = … }` уже после дефолтной инициализации.
>
> **В: Почему не сделать конструктор с параметрами?** О: Не требуется: события — простые контейнеры данных, инициализатор объекта читается яснее на месте публикации и не плодит перегрузок. Плюс единый стиль для всех ~30 событий.
>
> **В: Что будет, если забыть задать поле?** О: Оно останется значением по умолчанию (`null`/`0`/`false`/первый член enum). Компилятор не предупредит — это сознательный компромисс; критичные поля всегда задаются явно на месте публикации.
>
> **В: Почему struct, а не class?** О: Значимая семантика (нет `null`, копия на получателя), ограничение шины `where T : struct`, и отсутствие аллокаций в куче при частой рассылке.
>
> **В: Где гарантируется, что T — именно событие?** О: По соглашению (суффикс `Event`, папка `Events/`) и по ограничению `where T : struct` шины; жёсткого общего интерфейса-маркера у событий нет.
