# Регионная модель UI

> [!info] Назначение
> Модель управления видимостью панелей: панелей около десятка, одновременный показ перекрыл бы обзор. Каждая панель приписана к региону — группе взаимоисключающих панелей, где открытой может быть только одна. Исполняет [[PanelRegionRouter]], настраивает `NavBarConfig`, со стороны сцены — [[RegionMember]] и [[RegionNavButton]].

### Суть подхода

- ##### Регион
    - Группа панелей, внутри которой открыта максимум одна. При открытии новой прежний владелец региона скрывается; при закрытии регион не пустует — возвращается панель по умолчанию.
- ##### Конфигурация
    - Ассет `NavBarConfig`: на каждый модуль — идентификатор, перечень режимов приложения, где он доступен, имя региона, признак «по умолчанию». Связь «кнопка → конфигурация → панель» держится на совпадении строкового ключа.
- ##### Роли объектов
    - [[PanelRegionRouter]] (корневая область) — правила: открытие/закрытие/переключение, единственность, `RegionChangedEvent`.
    - [[RegionNavButton]] — кнопка навигации, хранит только строковый id модуля и просит маршрутизатор переключить его.
    - [[RegionMember]] — универсальная прослойка показа/скрытия на стороне сцены; при наличии делегирует особой реализации `IRegionSurface`.
- ##### Перестройка при смене режима
    - По `ModeChangedEvent` маршрутизатор закрывает недоступные в новом режиме панели, открывает панели по умолчанию в опустевших регионах и обновляет кнопки — состав интерфейса меняется без участия самих панелей.

### К защите

> [!question] Вероятные вопросы
> **В: Зачем кнопка хранит строку, а не ссылку на панель?** О: Развязка. Кнопка не знает о панели; общий ключ связывает кнопку, запись в `NavBarConfig` и зарегистрированную в маршрутизаторе поверхность. Панель можно заменить, не трогая кнопку.
>
> **В: Что если в регионе закрыли всё?** О: Регион не остаётся пустым — маршрутизатор открывает панель по умолчанию.
>
> **В: Как интерфейс перестраивается при смене режима?** О: Маршрутизатор ловит `ModeChangedEvent`, гасит недоступные панели, поднимает дефолтные и обновляет навигацию. Панели в этом не участвуют.
>
> **В: Регистрация при каждой сцене не дублирует записи?** О: Регистрация идемпотентна; записи, чей объект уничтожен вместе со сценой, маршрутизатор отбрасывает сам (проверка «живости» Unity-объекта).
