# Версионирование схем данных

> [!info] Назначение
> Механизм совместимости файлов разных версий. Каждый сериализуемый файл несёт поле `schemaVersion`; при чтении устаревшие схемы приводятся к актуальным. Применяется в [[SceneSerializer]] (сцена) и [[AnimationStorage]] (анимация).

### Суть подхода

- ##### Что такое схема
    - Схема — формат конкретного файла: состав полей структуры и правила записи на диск. У каждого типа данных **своя** схема и **независимая** нумерация: scene.json, animation.json и шаблон восстановления версионируются порознь.
- ##### Несопоставимость номеров
    - Текущая v3 схемы сцены означает лишь, что формат сцены пересматривался дважды; формат анимации при этом менялся один раз (v2), а шаблон восстановления — ни разу. Номера разных схем между собой **не сравнимы**.
- ##### Где живёт миграция
    - Приведение — инлайн на границе десериализации (в [[SceneSerializer]]`.Deserialize`), отдельного класса-мигратора нет. Номер версии пишется в сам файл, и при чтении приложение знает, по какой схеме файл сохранён.

### Два подхода к старым файлам

- ##### Аддитивный (сцена)
    - Схема сцены росла аддитивно: новые версии добавляли поля, не меняя смысла старых. Старый файл дополняется значениями по умолчанию (`v1→v2→v3` цепочкой `if`, не `else if`) — работа пользователя не искажается. v3 добавил позы костей (`BonePoses`).
- ##### Неразрушающий (анимация)
    - Для анимации выбран обратный принцип: файл с **неподдерживаемой** версией не приводится и **вообще не модифицируется** — чтобы не испортить данные, формат которых приложение не понимает.

### К защите

> [!question] Вероятные вопросы
> **В: Почему номера схем не совпадают?** О: Потому что они независимы — каждый файл версионируется отдельно, номер растёт только при изменении формата именно этого файла.
>
> **В: Что происходит со старой сценой при загрузке?** О: Десериализатор видит `schemaVersion`, по цепочке проверок дополняет недостающие поля значениями по умолчанию и приводит к v3 — прямо в момент чтения.
>
> **В: Почему сцену мигрируете, а анимацию нет?** О: Схема сцены развивалась аддитивно (дополнение безопасно). Для анимации безопаснее не трогать файл неизвестной версии вовсе, чем рискнуть исказить данные.
>
> **В: Зачем вообще версия в файле?** О: Без неё нельзя отличить старый формат от нового и нельзя безопасно эволюционировать формат, не ломая ранее сохранённые работы пользователя.
