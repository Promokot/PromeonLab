# События — анимация

> [!info] Назначение
> События транспорта, воспроизведения и правки анимации. `public struct` без собственных конструкторов (см. [[Событийные структуры (Event)]]), рассылка через [[EventBus]]. Папки `Animation/Events/`, `RigBuilder/Events/`.

### Транспорт и воспроизведение

- ##### FrameChangedEvent
    - Поля: `int Frame`. Конструирование: `new FrameChangedEvent { Frame = CurrentFrame }` (и `{ Frame = 0 }` при сбросе).
    - Публикует [[AnimationClock]] при смене **целого** кадра транспорта. Слушают [[AnimationAuthoring]] (сэмплирует клип) и [[AnimatorPanel]] (двигает ползунок). Важно: летит только при переходе через целую границу, плавность даёт `CurrentFrameContinuous` (дробное), а не это событие.
- ##### PlaybackStateChangedEvent
    - Поля: `bool IsPlaying`, `int Frame`, `bool Completed`. Конструирование: `new PlaybackStateChangedEvent { IsPlaying = false, Frame = 0, Completed = true }` (конец) или `{ IsPlaying = IsPlaying, Frame = CurrentFrame }`.
    - Публикует [[AnimationClock]] на Play/Pause/конце. `Completed = true` отличает естественное завершение (доигралось до конца) от обычной паузы — [[AnimatorPanel]] обновляет кнопку и ползунок.
- ##### LoopFrameChangedEvent
    - Поля: `string OwnerNodeId`, `int Frame`. Конструирование: `new LoopFrameChangedEvent { OwnerNodeId = owner, Frame = frame }`.
    - Только для отображения: фоновое зацикленное воспроизведение ([[AnimationPlaybackSampler]]) сообщает свой кадр, чтобы ползунок выбранного объекта следовал за ним. Намеренно отделено от `FrameChangedEvent`, привязанного к транспортным часам и сэмплированию клипа.

### Правка данных

- ##### AnimationContainerChangedEvent
    - Поля: `string OwnerNodeId`, `ContainerChange Change`. Конструирование: `new AnimationContainerChangedEvent { OwnerNodeId = ownerNodeId, Change = ContainerChange.Added }`.
    - Публикует [[AnimationAuthoring]] при изменении контейнера действия. `ContainerChange`: `Added, Removed, LengthChanged, FpsChanged, TracksChanged, InterpolationChanged, LoopChanged` — получатель ([[AnimatorPanel]]) по виду изменения решает, перестраивать дорожки или обновить точечно.
- ##### AnimationKeyframeChangedEvent
    - Поля: `string NodeId`, `string OwnerNodeId`, `int Frame`, `KeyframeChange Change`. `KeyframeChange`: `Added, Removed, Overwritten`.
    - Публикует [[AnimationAuthoring]] при постановке/удалении/перезаписи ключа. Несёт и `NodeId` (чья дорожка — объект или кость), и `OwnerNodeId` (чей контейнер), и кадр — этого хватает UI, чтобы обновить нужную дорожку без полной перестройки.
- ##### BonesVisibilityChangedEvent
    - Поля: `string RigNodeId`, `bool Visible`. Конструирование: `new BonesVisibilityChangedEvent { RigNodeId = rigNodeId, Visible = on }`.
    - Публикуется при включении/выключении режима костей рига; [[AnimatorPanel]]/[[OutlinerPanel]] показывают/скрывают дорожки и строки костей этого рига.

### К защите

> [!question] Вероятные вопросы
> **В: Почему FrameChangedEvent и LoopFrameChangedEvent — разные события?** О: Первое привязано к транспортным часам и управляет сэмплированием клипа выбранного объекта; второе — чисто отображательное, от фонового зацикленного воспроизведения других объектов. Смешивать их нельзя, иначе фоновые циклы дёргали бы транспорт.
>
> **В: Зачем в PlaybackStateChangedEvent поле Completed?** О: Чтобы отличить «доиграл до конца» от «поставили на паузу» — UI по-разному реагирует (сброс ползунка к началу против остановки на месте).
>
> **В: Почему в событиях правки передаёте enum-вид изменения, а не отдельные события?** О: Один тип события + поле `ContainerChange`/`KeyframeChange` компактнее, чем десяток типов; получатель ветвится по значению и выбирает минимальное обновление UI.
>
> **В: Как событие конструируется?** О: Инициализатором объекта на месте публикации; своих конструкторов нет (см. [[Событийные структуры (Event)]]). Незаданный enum по умолчанию равен своему нулевому члену (`ContainerChange.Added`, `KeyframeChange.Added`).
