> [!info] Назначение
> `RigDefinition` и `BoneRecord` — сериализуемые структуры описания скелета ассета. Хранятся в `AssetEntityRecipe.rig`; сериализуются `JsonUtility` как часть `imported-lib.json`. Публичные поля — требование `JsonUtility`. Листинг 3.40.

### Обзор

##### Роль и место

`[Serializable]` классы (не `MonoBehaviour`, не `ScriptableObject`). `RigDefinition` несёт полное описание скелета; `BoneRecord` — запись одной кости. Заполняются `RigDefinitionExtractor` при импорте, читаются `RigEntityFabricator` при восстановлении.

##### Ключевые поля

| Поле | Тип | Назначение |
|---|---|---|
| `RigDefinition.SchemaVersion` | `int = 1` | версия схемы для будущих миграций |
| `RigDefinition.AssetId` | `string` | идентификатор ассета (имя корневого GO при извлечении) |
| `RigDefinition.TerminalBonesAxis` | `TerminalBoneAxis` | ось ромба концевых костей (Auto/X/Y/Z) |
| `RigDefinition.InvertTerminalBonesAxis` | `bool` | инверсия выбранной оси |
| `RigDefinition.Bones` | `List<BoneRecord>` | перечень костей |
| `RigDefinition.IkChains` | `List<IkChainRecord>` | IK-цепи (сериализованы, солвер не реализован) |
| `BoneRecord.BoneName` | `string` | имя кости (ключ для словаря `ProxyRigRuntime._boneProxies`) |
| `BoneRecord.TranslationLocked` | `bool = true` | блокировка перемещения кости (по умолчанию включена) |

### Разбор кода

##### RigDefinition — публичные поля под JsonUtility

```csharp
[Serializable]
public class RigDefinition
{
    public int SchemaVersion = 1;
    public string AssetId;
    public TerminalBoneAxis TerminalBonesAxis;
    public bool             InvertTerminalBonesAxis;
    public List<BoneRecord> Bones = new();
    public List<IkChainRecord> IkChains = new();
}
```

> `JsonUtility` сериализует только **публичные** поля или приватные с `[SerializeField]`. Для чистых data-классов без поведения проект допускает `public` поля (см. CLAUDE.md: «plain `[Serializable]` data classes … may use `public` fields»). `SchemaVersion = 1` инлайн — поле будет в JSON с правильным значением по умолчанию при создании новой записи без явной инициализации.

##### BoneRecord.TranslationLocked — реальное поле, не из нарратива ВКР

```csharp
[Serializable]
public class BoneRecord
{
    public string BoneName;
    public bool TranslationLocked = true;
}
```

> В нарративе ВКР (Листинг 3.40) `BoneRecord` показан только с `BoneName`. В реальном исходнике есть также `TranslationLocked = true` — флаг блокировки перемещения кости (по умолчанию кости не переводятся, только вращаются). Нарратив упрощён для компактности.

##### IkChains — сериализованы, но солвер не реализован

```csharp
public List<IkChainRecord> IkChains = new();
```

> По `CLAUDE.md`: «IK chains are serialized but no solver consumes them yet». Поле присутствует в `imported-lib.json`, но при восстановлении никакой компонент его не читает. Это «задел» для будущей реализации IK без миграции схемы — схема уже поддерживает цепи, нужно только написать солвер.

### К защите

##### Вероятные вопросы

> [!question]
> **В:** Почему поля `public`, если в проекте правило «`[SerializeField] private`»?
> **О:** Правило `[SerializeField] private` относится к `MonoBehaviour` и `ScriptableObject`. `RigDefinition` — plain `[Serializable]` data-класс, сериализуемый `JsonUtility`. `JsonUtility` не видит приватные поля без `[SerializeField]`, а добавлять `[SerializeField]` в каждое поле data-структуры — избыточный шум. Проект явно делает исключение для таких классов в `CLAUDE.md`.

> [!question]
> **В:** Что произойдёт при загрузке старого `imported-lib.json` без поля `TranslationLocked`?
> **О:** `JsonUtility` не трогает поля, отсутствующие в JSON: `TranslationLocked` получит значение по умолчанию из инициализатора поля — `true`. Это правильное поведение: старые записи молча получат «locked», что соответствует ожидаемому начальному состоянию.

> [!question]
> **В:** Зачем `BoneName` как ключ, а не индекс в массиве `SkinnedMeshRenderer.bones`?
> **О:** Индексы нестабильны: при обновлении модели порядок костей может измениться. Имя кости — семантически стабильный идентификатор, назначенный художником. `ProxyRigRuntime._boneProxies` хранит `Dictionary<string, Transform>` — поиск по имени, неизвестные имена пропускаются.
