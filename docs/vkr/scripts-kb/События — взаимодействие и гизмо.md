# События — взаимодействие и гизмо

> [!info] Назначение
> События выделения, навигации UI, фокуса клавиатуры и работы гизмо. `public struct` без собственных конструкторов (см. [[Событийные структуры (Event)]]), рассылка через [[EventBus]]. Папки `SceneComposition/Events/`, `SpatialUi/Events/`, `VrInteraction/Events/`.

### Выделение и навигация

- ##### SelectionChangedEvent
    - Поля: `string SelectedNodeId` (пустая строка/`null` = ничего не выбрано). Конструирование: `new SelectionChangedEvent { SelectedNodeId = _selectedNodeId }`.
    - Публикует [[SelectionManager]]. Слушают [[InspectorPanel]], [[GizmoDriver]], [[ProxyRigRuntime]], [[SelectionVisualSync]] — обновляют свойства, гизмо, подсветку кости и обводку. Одно из самых «центральных» событий.
- ##### SceneSelectedEvent
    - Поля: `string SceneId`, `string DisplayName`. Конструирование: пустое `new SceneSelectedEvent { SceneId = string.Empty, DisplayName = string.Empty }` (сброс) или заполненное по выбранной строке.
    - Публикует [[ScenePickerPanel]] при выборе строки в списке сцен; слушает [[MainMenuPanel]] — активирует кнопку открытия и дополняет её подпись именем сцены. (Не путать с `SelectionChangedEvent` — это про выбор сцены в меню, а не объекта в сцене.)
- ##### RegionChangedEvent
    - Поля: `string RegionKey`, `string OpenModuleId` (`null`, если регион опустел). Конструирование: `new RegionChangedEvent { RegionKey = region, OpenModuleId = moduleId }`.
    - Публикует [[PanelRegionRouter]] при каждой смене открытой панели в регионе; кнопки навигации обновляют подсветку активной. См. [[Регионная модель UI]].
- ##### KeyboardFocusEvent
    - Поля: `TMP_InputField Target`. Конструирование: `new KeyboardFocusEvent { Target = _field }`.
    - Публикует посредник [[VrInputFieldFocusBridge]] при тапе по полю ввода; слушает экранная клавиатура — запоминает активное поле. Поле и клавиатура не ссылаются друг на друга напрямую.

### Гизмо

- ##### GizmoDragStartedEvent / GizmoDragEndedEvent
    - Поля у обоих: `string TargetNodeId`. Конструирование: `new GizmoDragStartedEvent { TargetNodeId = _targetNodeId }`.
    - Публикует [[GizmoDragSession]]/[[GizmoDriver]] на старте и завершении перетаскивания. Позволяют, например, [[AnimatorPanel]] или трекеру изменений реагировать на границы манипуляции.
- ##### GizmoModeChangedEvent
    - Поля: `GizmoMode Mode` (`Move, Rotate, Scale`). Сообщает о переключении инструмента гизмо; панель инструментов обновляет активную кнопку.
- ##### GizmoToolsPanelOpenedEvent / GizmoToolsPanelClosedEvent
    - Маркеры без полей: `public struct GizmoToolsPanelClosedEvent { }`. Сообщают об открытии/закрытии панели инструментов гизмо — например, [[GizmoDriver]] проверяет отсутствие активного перетаскивания перед закрытием.

### К защите

> [!question] Вероятные вопросы
> **В: Как отличить «ничего не выбрано»?** О: `SelectedNodeId` — пустая строка/`null`. Получатели проверяют это и гасят свойства/гизмо/обводку (см. [[EmptySpaceClickDeselector]]).
>
> **В: Чем SceneSelectedEvent отличается от SelectionChangedEvent?** О: Первое — выбор **сцены** в главном меню (id + отображаемое имя), второе — выбор **объекта** внутри загруженной сцены. Разные подсистемы, разные получатели.
>
> **В: Почему поле ввода и клавиатура связаны событием, а не ссылкой?** О: Чтобы добавлять новые поля в любые панели без правок клавиатуры — достаточно повесить посредник [[VrInputFieldFocusBridge]], который опубликует `KeyboardFocusEvent`.
>
> **В: KeyboardFocusEvent несёт ссылку на TMP_InputField — это нормально для struct-события?** О: Да: само событие — значимый тип, а поле — ссылка на UI-объект. Копируется ссылка; клавиатура лишь запоминает, в какое поле направлять ввод.
>
> **В: Как конструируются эти события?** О: Инициализатором объекта на месте публикации; конструкторов нет (см. [[Событийные структуры (Event)]]). Незаданный `OpenModuleId`/`SelectedNodeId` остаётся `null`.
