# События — ассеты и импорт

> [!info] Назначение
> Конвейер импорта модели и спавна ассета, выстроенный целиком на событиях. `public struct` без собственных конструкторов (см. [[Событийные структуры (Event)]]), рассылка через [[EventBus]]. Папки `SpatialUi/Events/`, `AssetBrowser/Events/`.

### Цепочка импорта

- ##### FilePickedEvent
    - Поля: `string Path`. Публикуется файловым диалогом при выборе файла; слушает [[ImportPipeline]] — выбирает `IAssetImporter` по расширению и открывает мастер.
- ##### ImportRequestedEvent
    - Поля: `string FilePath`, `string SuggestedName`, `AssetType SuggestedType`. Конструирование: `new ImportRequestedEvent { FilePath = …, SuggestedName = …, SuggestedType = … }`.
    - Направление **конвейер → мастер**: [[ImportPipeline]] просит [[ImportWizardPanel]] показать форму с предложенными именем и типом. `AssetType`: `Object, Rig, Reference`.
- ##### ImportConfirmedEvent
    - Поля: `bool Confirmed`, `string FilePath`, `string DisplayName`, `AssetType ChosenType`, `TerminalBoneAxis TerminalBonesAxis`, `bool InvertTerminalBonesAxis`. Конструирование (отмена): `new ImportConfirmedEvent { Confirmed = false, FilePath = _filePath }`.
    - Направление **мастер → конвейер**: `Confirmed = false` означает отмену. При отмене заполнен только путь — остальные поля остаются по умолчанию (`ChosenType = AssetType.Object`, `TerminalBonesAxis = TerminalBoneAxis.Auto`, т.к. это нулевые члены enum). [[ImportPipeline]] копирует источник и пишет запись библиотеки.
- ##### AssetImportedEvent
    - Поля: `string AssetId`. Публикуется после записи ассета; слушает браузер ([[AssetBrowserPanel]]) — обновляет сетку миниатюр.

### Спавн

- ##### AssetSpawnRequestedEvent
    - Поля: `ILabAsset Asset`, `Vector3 Position`, `Quaternion Rotation`. Конструирование (многострочное): `new AssetSpawnRequestedEvent { Asset = _selectedAsset, Position = pos, Rotation = … }`.
    - Публикует [[AssetBrowserPanel]] при выборе ассета; слушает [[AssetSpawner]] — восстанавливает объект через [[AssetEntityBuilderRegistry]]. Единственное событие группы, несущее ссылочный тип (`ILabAsset`) и Unity-структуры.

### К защите

> [!question] Вероятные вопросы
> **В: Почему импорт построен на событиях, а не на прямых вызовах?** О: Чтобы развязать диалог файла, конвейер и мастер: каждый шаг публикует результат, следующий подхватывает. Можно заменить любой из них, не трогая соседей (см. [[Паттерн Publish-Subscribe]]).
>
> **В: Как отменяется импорт?** О: Мастер публикует `ImportConfirmedEvent` с `Confirmed = false`; конвейер видит флаг и прекращает обработку. Незаполненные поля при этом — значения по умолчанию.
>
> **В: AssetSpawnRequestedEvent несёт ILabAsset — это ссылка, не копия. Не опасно?** О: Поле — ссылка на ассет-описание; копируется ссылка, а не объект. Получатель только читает описание для восстановления, не мутирует его.
>
> **В: Почему два события на импорт — Requested и Confirmed?** О: Между ними — взаимодействие пользователя в мастере. `Requested` открывает форму с предложением, `Confirmed` возвращает выбор пользователя (или отмену).
